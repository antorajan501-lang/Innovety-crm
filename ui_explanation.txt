================================================================================
                    MRF CRM MODERNIZED UI/UX SYSTEM EXPLANATION
================================================================================

This document explains the design principles, visual styling, custom animations, 
and development breakdown for the modernized MRF Enterprise CRM dashboard. 

The user interface was redesigned and optimized over a 50-day engineering sprint 
by a dedicated two-person development team.

--------------------------------------------------------------------------------
1. DESIGN THEME & VISUAL PALETTE
--------------------------------------------------------------------------------
The CRM features a modernized "Emerald Green & Mint" corporate theme. 
Instead of basic gray or dark blue tones, we use a sleek Emerald Green, Mint, 
and Cyber Mint color system.

Visual Tokens:
- Primary Color: Emerald (#10b981) - used for active status indicators, 
  sidebar selection highlights, and main action buttons.
- Secondary Color: Mint (#34d399) - used for task category badges, 
  tickets desk, and sub-panels.
- Backgrounds: Very light mint-slate gray (#f4f9f6) in light mode, and a rich dark 
  emerald slate (#06140f) in dark mode.
- Typography: Outfit & Plus Jakarta Sans fonts are integrated to provide a 
  geometric, clean look suited for corporate applications.
- Additional Accent Mappings:
  * Info: Sky Blue (#38bdf8) - for notifications & informational logs.
  * Indigo: Deep Indigo (#6366f1) - for cards and visual highlights.
  * Violet: Purple (#8b5cf6) - for ticketing desk & review requests.
  * Amber: Warning Yellow (#f59e0b) - for warning statuses.
  * Rose: Issue Red (#f43f5e) - for urgent priority tickets.

--------------------------------------------------------------------------------
2. PREMIUM UI UTILITIES & GLASSMORPHISM
--------------------------------------------------------------------------------
- Glassmorphism Panels: UI blocks use backdrop blur filters (24px) combined with 
  translucent borders and ambient drop shadows to define visual depth.
- Smooth Inputs: All input boxes, drop-downs, and forms have rounded corner profiles 
  (rounded-xl) and display custom glowing outlines on focus.
- Micro-Animations: Hover states translate upward slightly (-3px) and active 
  buttons scale down slightly (95% size) to simulate responsive tactile clicks.

--------------------------------------------------------------------------------
3. ANIMATION DETAILS
--------------------------------------------------------------------------------
- Session Tracker: The dashboard's active WebSocket connection features a pulsing 
  green beacon (pulse-active) showing real-time presence sync.
- Modal Zoom: Confirmation and prompt dialogs transition smoothly using a 95% zoom-in 
  effect combined with a background blur overlay (black/60).
- Welcome Banner: The greeting banner uses a subtle teal-cyan gradient with 
  semi-transparent borders.

--------------------------------------------------------------------------------
4. ENGINEERING WORK DISTRIBUTION (50-DAY PLAN)
--------------------------------------------------------------------------------
The project architecture was split between two developers over a 50-day sprint:

Developer A: Backend & API (Days 1 - 50)
- Days 01 - 15: Schema creation, Prisma migrations, and database modeling.
- Days 16 - 25: JWT security pipeline, session middleware, and user management.
- Days 26 - 38: WebSocket (Socket.io) real-time presence connection routing.
- Days 39 - 50: PDF/CSV exports, reports filters, and activities logs compilation.

Developer B: Frontend & UX (Days 1 - 50)
- Days 01 - 14: Theme configuration, layout layouting, and sidebar menus.
- Days 15 - 28: Drag-and-drop Kanban Board, Backlog tab, and Timeline Gantt chart.
- Days 29 - 40: Geolocation and OpenStreetMap mapping API check-ins.
- Days 41 - 50: Custom Modal prompts (confirmations) and styling optimizations.
